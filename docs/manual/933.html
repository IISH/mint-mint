Assign PIDs

Generate pid with extension function

During the mapping phase you can assign a persistent identifier to an element, based on a naming authority, a local identifier and a resolve URL.

First, we drop the element which contains the resolve url to the place, where we want to create our persistent identifier. In the example above, we mapped the value of europeana:isShownAt, (which contains a URL pointing to a webpage) to the edm:isShownAt@rdf:resource target attribute.

To register the pid now we can click on the function icon next to the mapped field.

To use our custom function, first we should select 'Custom function' from the Function type selector dropdown. To register PIDs, we will use the 'getQuickPidReqest' xsl function from the 'ialhi' namespace.

This function have three parameters:

  1. Naming Authority
  2. Local Identifier
  3. Resolve URL

In the example above the values for this element are the following:

  1. Naming Authority: 12345
  2. Local Identifier: The dc:identifier element
  3. Resolve URL: The actual edm:isShownAt link (the current element)

With these parameters in mind, you should call the extension function like this: ialhi:getQuickPidRequest('12345', //dc:identifier, .)

Now every time a mapping will be invoked (upon preview or tranformation) this function ensures, that a pid will be written in the element text.

PIDs in EDM

Based on convention in the final EDM PID referring to isShownAt page will be used to generate the URI for the providedCHO and aggregation element. This (at the moment) can be achieved with a chained transformation.

First let's transform our dataset with the basic mapping, which already contains the PID function call. (In the example this is called ASKI2EDM.)

When the procedure is ready select the transformation.

In the selected transformation, select the option to transform. This is how you can create a chained tranformation (when an already transformed dataset resultset will be transformed based on a different mapping.)

There is an xsl based mapping available (in the example it is called edm2edm_pid) which copies the isShownAt element and puts its value (concatenated with '#cho' or '#aggregation') to providedCHOs and aggregations rdf:resource attribute. Select this mapping and run the transformation.

Run the process and in the resultset window you can check the modified values.

After this, all you have to do is publish your dataset. Upon publish it will go through the same chained transformation procedure.
